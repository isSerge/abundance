Expand description

Metadata for smart contact methods.

Metadata encoding consists of this enum variant treated as u8 followed by optional metadata encoding rules specific to metadata type variant (see variantâ€™s description).

This metadata is enough to fully reconstruct the hierarchy of the type to generate language bindings, auto-generate UI forms, etc.

Init = 2

#[init] method.

Initializers are encoded af follows:

  • Length of method name in bytes (u8)
  • Method name as UTF-8 bytes
  • Number of named arguments (u8, excluding state argument &self or &mut self, but including return type that is not () or Result<(), ContactError>)

Each argument is encoded as follows:

  • Argument type as u8, one of:
    • Self::EnvRo
    • Self::EnvRw
    • Self::TmpRo
    • Self::TmpRw
    • Self::SlotRo
    • Self::SlotRw
    • Self::Input
    • Self::Output
  • Length of the argument name in bytes (u8, except for Self::EnvRo and Self::EnvRw)
  • Argument name as UTF-8 bytes (except for Self::EnvRo and Self::EnvRw)
  • Only for Self::Input and Self::Output recursive metadata of argumentâ€™s type as described in IoTypeMetadataKind with following exception:
    • For last Self::Output this is skipped if method is Self::Init (since it is statically known to be Self) and present otherwise

NOTE: Self::Output, regardless of whether it is a return type or explicit #[output] argument is encoded as a separate argument and counts towards number of arguments. At the same time, self doesnâ€™t count towards the number of arguments as it is implicitly defined by the variant of this struct.

pub const fn compact(metadata: &[u8]) -> Option<([u8; 8192], usize)>

Produce compact metadata.

Compact metadata retains the shape, but throws some details. Specifically following transformations are applied to metadata (crucially, method names are retained!):

  • Struct, trait, enum and enum variant names removed (replaced with 0 bytes names)
  • Structs and enum variants turned into tuple variants (removing field names)
  • Method argument names removed (removing argument names)

This means that two methods with different argument names or struct field names, but the same shape otherwise are considered identical, allowing for limited future refactoring opportunities without changing compact metadata shape, which is important for MethodFingerprint.

Returns None if input is invalid or too long.
